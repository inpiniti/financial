# guru-report 스킬 토큰 절감 업그레이드

> 날짜: 2026-07-11 | 상태: 적용 완료 (`.claude/skills/guru-report/SKILL.md`, `references/data-sources.md`)

## 배경과 목표

guru-report 스킬은 티커 하나당 ① 데이터 팩 → ② 13인 병렬 보고서 → ③ 원탁 토론 + 최종 보고서 파이프라인을 돌렸다. 토큰 소모가 심해 Fable 모델에서는 종목 한두 개 분석이면 사용 한도가 바닥나는 문제가 있었다.

**목표: 토론 제거 + 개별/최종 보고서 퀄리티 유지 + 토큰 소모 대폭 절감.**

## 토큰 소모 3대 주범 (진단)

1. **2단계 웹 조사** — "데이터 팩에 없으면 웹에서 조사하라"는 지시 때문에 13개 에이전트가 각자 WebSearch/WebFetch를 수행. 에이전트당 가변 수만 토큰.
2. **3단계 토론** — 진행자가 13개 보고서 전문(~130KB)을 입력으로 읽고, 토론 전문(실측 `V_토론.md` 14KB / `DIS_토론.md` 24KB 한국어 출력) + 최종보고서를 생성.
3. **모델 단가** — 14개 에이전트 전부가 세션 모델(Fable)로 실행됨.

## 변경 내용

### 1단계 — 데이터 팩 강화 (수집은 여전히 1회)

2단계 웹 조사를 금지하는 대신, 페르소나 체크리스트들이 요구하는 항목을 데이터 팩 필수 목록에 추가:

- **재무 안전성**: 유동자산·유동부채·유동비율, 총부채, 재고, 이자보상배율 (그레이엄 NCAV·안전성 평가용)
- **수급·심리**: 공매도 비율(short % of float — 버리), 내부자·기관 지분율(피셔·린치), 애널리스트 등급 분포·목표가(템플턴·뉴욕주민)
- **정성 리스크**: 소송·규제·회계 이슈 요약(버리·클라먼), 차기 실적 발표일 등 예정 이벤트
- **기술적 요약**: 52주 위치, 최근 급등락 이벤트와 원인(코스톨라니·슈웨거)

`references/data-sources.md`에 수집 레시피 추가 (전부 기존 Yahoo 무료 엔드포인트 확장, 신규 인증 불필요):
- quoteSummary 모듈: `majorHoldersBreakdown`, `recommendationTrend`, `calendarEvents`, `defaultKeyStatistics`의 `shortPercentOfFloat`/`heldPercentInsiders`/`heldPercentInstitutions`
- fundamentals-timeseries 타입: `annualCurrentAssets`, `annualCurrentLiabilities`, `annualInventory`

### 2단계 — 13인 병렬 보고서 (Sonnet, 웹 금지, 구조화 요약 반환)

- Agent 호출 시 **`model: "sonnet"` 지정** — Fable 사용량 소모의 대부분이 여기서 제거됨. 개별 보고서는 "페르소나 파일 + 데이터 팩"을 체크리스트대로 적용하는 작업이라 Sonnet으로도 퀄리티가 유지됨. 최종 종합만 세션 모델(Fable)이 담당.
- **웹 조사 전면 금지**: "WebSearch/WebFetch/Bash 등 외부 조사 도구 사용 금지. 읽어도 되는 것은 페르소나 파일과 데이터 팩 두 개뿐. 없는 수치는 '확인 불가'로 남기고 불확실성을 판단에 반영."
- 개별 보고서 템플릿(한 줄 결론/체크리스트 평가/밸류에이션/리스크/최종 의견)은 **그대로 유지** — 개별 보고서 퀄리티 불변.
- 최종 응답을 기존 "한 줄" → **구조화 요약 블록(~12줄)**으로 확장: 인물/의견/확신도 + 핵심 논거 2~3개 + 적정가·매수 가격대 + 트리거·재검토 조건 + 대표 발언 1문장.
- 메인 루프가 13개 요약 블록을 모아 `docs/report/{날짜}/_data/{티커}_요약.md`로 저장.

### 3단계 — 토론 제거, 종합 에이전트로 교체

- 토론 진행자 + `{티커}_토론.md` 산출물 **완전 삭제**.
- 대신 **종합 에이전트 1개**(세션 모델)가:
  1. 필독: `_data/{티커}_요약.md` + `_data/{티커}.md`
  2. 선택 정독: 요약을 훑은 뒤 대표 강세 1~2인 + 대표 약세·관망 1~2인 + 논거가 독특한 1인, **최대 4개 보고서만** 전문을 읽어 인용·논거 보강 (13개 전부 읽기 금지)
  3. 산출: `최종/{티커}_최종보고서.md` **하나만**
- 최종보고서 템플릿은 현행 유지(종합 결론/강세론 핵심/약세론 핵심/미해결 쟁점/실행 가이드/13인 요약표) — 토론의 가치(논거 대립·표결·트리거)는 이 구조가 전부 보존한다. 표결 집계는 요약 파일에서 직접 계산.

### 유지되는 것

- 페르소나 파일 13개 (`references/personas/*.md`) — 말투·체크리스트 퀄리티의 원천, 무수정
- 개별 보고서 템플릿과 분량
- 최종보고서의 6개 섹션 구조
- 페르소나 재생성 워크플로
