# 설계 v1 — 투자 보고서 뷰어 (Report Viewer)

> 작성일: 2026-07-10 | 상태: 초안 (승인 대기) | 이 문서는 v1이며 이후 변경은 v2로 버전업한다 (v1은 얼림)

## 1. 목표

`docs/report/` 아래에 guru-report 스킬이 쌓는 마크다운 보고서(거장 13인 × 티커 × 날짜 + 토론 + 최종)를
브라우저 UI로 탐색·열람한다. 백엔드 없이 로컬에서 `npm run dev` 한 번으로 뜨는 정적 뷰어가 1차 목표.

## 2. 기술 스택 (확정)

| 항목 | 선택 | 비고 |
|---|---|---|
| 프레임워크 | **Vite + React 19 + TypeScript** | SPA, 백엔드 없음 |
| 스타일 | **Tailwind v4 + shadcn/ui** | **base color: neutral, style: mira** (사용자 지정) — init 시 shadcn 최신 문서에서 mira 스타일 플래그 확인 후 적용 |
| 라우팅 | react-router | 해시 라우팅 불필요, BrowserRouter |
| 마크다운 렌더 | react-markdown + remark-gfm | 보고서에 GFM 표가 많음 — 표 렌더 필수 |
| 데이터 로딩 | `import.meta.glob('../docs/**/*.md', { query: '?raw' })` | 빌드타임 번들. 새 보고서는 dev 서버가 glob HMR로 자동 반영 |

## 3. 디렉터리 구조

```
financial/
├── docs/                  ← 기존 (데이터 원본, 앱은 읽기만 한다)

│   ├── books/*.md
```

## 4. 데이터 계약 (핵심)

### 4-1. 경로 → 카탈로그

`docs/report/{date}/{author}/{file}.md`를 파싱해 아래 구조를 만든다.

```ts
type Verdict = '매수' | '보유' | '매도' | '관망'

interface GuruReport { guru: string; verdict: Verdict | null; confidence: number | null; raw: string }

interface TickerReport {
  date: string            // '2026-07-10'
  ticker: string          // '005930'
  name: string            // '삼성전자' ← interest ticker.md 매핑, 없으면 ticker 그대로
  gurus: GuruReport[]     // 거장 13인 (폴더명 = 거장명, '-' → ' ')
  data?: string           // _data/{ticker}.md
  final?: string          // 최종/{ticker}_최종보고서.md
  debate?: string         // 최종/{ticker}_토론.md
  status: 'final' | 'in-progress'   // final 존재 여부
  tally?: { 매수: number; 보유: number; 매도: number; 관망: number }  // 최종 헤더의 표결
  finalVerdict?: string   // 최종 헤더의 종합 의견 원문
}

type ReportCatalog = Map<date, TickerReport[]>
```

### 4-2. 메타데이터 파싱 규칙 (실측 포맷 기준)

- 거장 보고서 2행: `> 날짜: 2026-07-10 | 투자의견: 관망 | 확신도: 8/10`
  - 정규식: `투자의견:\s*(매수|보유|매도|관망)` / `확신도:\s*(\d+)\/10`
- 최종보고서 2행: `> 날짜: … | 종합 의견: **…** | 표결: 매수 1·보유 1·매도 6·관망 5`
  - 표결: `(매수|보유|매도|관망)\s*(\d+)` 전역 매칭
- **파싱 실패는 에러가 아니라 null** — 배지 없이 본문만 보여준다 (포맷 변동 내성)
- `interest ticker.md` 행 포맷: `{TICKER} : {한글명}` (예: `005930 : 삼성전자`, `BRK.B : 버크셔 해서웨이` — 티커에 `.` 허용)

## 5. 화면 구조

### 라우트

| 라우트 | 화면 | 내용 |
|---|---|---|
| `/` | 대시보드 | 날짜별 티커 카드 그리드. 카드 = 티커·이름·상태(최종/진행중)·종합의견·표결 미니바 |
| `/r/:date/:ticker` | 보고서 뷰 | 좌: 문서 목차 사이드바(최종/토론/거장13/데이터팩), 우: 마크다운 본문 |
| `/books` | 서재 | docs/books 목록 + 열람 (같은 MarkdownView 재사용) |


```
┌──────────────┬─────────────────────────────────────┐
│ 005930 삼성전자│  # 005930 최종 투자 보고서            │

│ ─────────────│                                      │
│ ★ 최종 보고서 │  ## 종합 결론                         │
│ 💬 원탁 토론  │  13인 표결의 단순 다수는…             │
│ ─ 거장 13인 ─ │                                      │
│ 워런 버핏 [관망8]                                     │
│ 벤저민 그레이엄 [매도8]                                │
│ …             │                                      │
│ ─────────────│                                      │
│ 📊 데이터 팩  │                                      │
└──────────────┴─────────────────────────────────────┘
```

- 사이드바 거장 항목에 의견 배지 + 확신도 표시 (매수=긍정색, 매도=부정색, 관망/보유=중립 — mira 팔레트 내에서)
- 진행중(최종 없음) 티커는 최종/토론 항목을 비활성 표기

## 6. 하지 않는 것 (v1 범위 제외)

- ❌ 백엔드/DB/배포 — 로컬 dev 서버만. 배포가 필요해지면 v2에서 정적 빌드+호스팅 검토
- ❌ 보고서 생성/편집 — 뷰어는 읽기 전용, 생성은 guru-report 스킬의 일
- ❌ 티커 간 비교 차트·시계열 분석 — 날짜가 쌓인 뒤 v2 후보
- ❌ 검색(전문 검색) — v1은 탐색 구조로 충분, 필요 시 v2

## 7. 리스크

| 리스크 | 대응 |
|---|---|
| shadcn `mira` 스타일 플래그가 예상과 다름 | 스캐폴딩 단계에서 shadcn 공식 문서 확인 후 적용, 부재 시 사용자에게 보고 후 대안 협의 |
| 보고서 헤더 포맷이 guru-report 버전업으로 변동 | 파싱 실패 = null 원칙 (본문 열람은 항상 가능), 파서를 meta.ts 한 파일에 격리 |
| `import.meta.glob`이 docs가 앱 루트 밖이라 미동작 | Vite `server.fs.allow`에 `../docs` 추가, glob 경로는 상대경로 — 스캐폴딩 단계에서 스모크 테스트 |
| 보고서 수 증가로 번들 비대 | glob lazy 모드(동적 import)로 본문은 열람 시 로딩, 메타만 eager |
