# Portfolio Advisor — 실계좌 기반 포트폴리오 조언

한국투자증권 API로 **실제 잔고**를 조회하고, guru-report 최종 보고서를 근거로 종목별 조언을 만든다.
파이프라인: **① 잔고 조회(스크립트) → ② 보고서 매칭 → ③ 분석·조언 작성 → ④ 저장·보고**

현재 범위는 **해외주식 잔고만**이다 (국내주식 잔고 API 문서가 추가되면 확장).

## 1단계 — 잔고 조회

```bash
node .claude/skills/portfolio-advisor/scripts/fetch_balance.mjs
```

- 저장소 루트의 `.env`에서 `KIS_APPKEY`, `KIS_APPSECRET`, `KIS_CANO`, `KIS_ACNT_PRDT_CD`를 읽는다. 없으면 스크립트가 안내 메시지와 함께 실패한다 → 사용자에게 `.env` 작성을 요청하고 중단.
- 접근토큰은 루트 `.kis-token.json`에 자동 캐시된다 (24h 유효). 토큰·앱키·시크릿 값을 **어떤 산출물이나 대화에도 절대 출력하지 마라.**
- 출력(JSON): `holdings[]` (ticker, name, qty, avg_price, cur_price, pl_rate_pct, weight_pct 등), `cash[]`, `summary_krw` (총자산·총평가손익 등).
- API 실패 시(rt_cd≠0) 에러 메시지를 사용자에게 그대로 전달하고 중단한다. 임의로 잔고를 추정하지 마라.
- 참고: 미국장 마감 후 현재가는 종가 기준이고, 장 종료 직후 30분가량은 반영 지연이 있을 수 있다 (API 명세의 유의사항).

## 2단계 — guru-report 보고서 매칭

- Glob: `docs/report/*/최종/*_최종보고서.md` → 파일명에서 티커 추출, 같은 티커가 여러 날짜에 있으면 **가장 최근 날짜만** 사용.
- 보유 종목별로 분류:
  - **보고서 있음** → 해당 최종 보고서를 읽고 종합 의견(매수/보유/매도/관망, 확신도, 진입가격대)을 추출
  - **보고서 없음 또는 30일 초과 경과** → "신규/재분석 필요" 목록에 넣는다 (이 스킬이 guru-report를 직접 실행하지는 않는다 — 권고만)
- 보고서가 하나도 없어도 중단하지 않는다. 계좌 데이터만으로 3단계의 구조 분석은 수행한다.

## 3단계 — 분석·조언 작성

분석은 두 층으로 구성한다.

**A. 포트폴리오 구조 (계좌 데이터만으로 항상 수행)**
- 종목별 비중(weight_pct) 상위 집중도 — 상위 1종목/3종목 비중
- 섹터·테마 쏠림 (예: 반도체 계열 합산 비중)
- 손익 현황 — 큰 평가손실(-20% 이하)·큰 평가이익(+50% 이상) 종목은 반드시 언급
- 현금(예수금) 비중

**B. 종목별 조언 (보고서 연계)**
각 보유 종목을 4분류하고, 분류 근거로 **어느 보고서의 어떤 논거인지 인용**한다:

| 분류 | 기준 (예시) |
|---|---|
| 추가 매수 후보 | 보고서 종합의견 매수 + 현재가가 보고서 진입가격대 안/아래 |
| 유지 | 보고서 보유 의견, 또는 매수 의견이지만 이미 비중 과다 |
| 매도/축소 검토 | 보고서 매도·관망 의견, 또는 비중 과다 + 확신도 낮음 |
| 신규/재분석 필요 | 최신 보고서 없음 → `guru-report {티커}` 실행 권고 |

- 보고서 의견과 기계적으로 일치시킬 필요는 없다. 비중·손익·집중도라는 계좌 맥락을 더해 판단하되, 보고서와 다른 결론을 낼 때는 이유를 명시한다.

**주수 환산 (추가 매수·매도/축소 분류에 필수)**
비중 제안 옆에 반드시 "몇 주를 사고/팔면 되는지"를 함께 표기한다:
- 기준 총액 = 주식 평가액 합계 (weight_pct와 같은 기준). 환산은 각 종목의 외화 평가액 × 기준환율로 통일.
- **추가 매수**: 필요 주수 ≈ (목표비중 − 현재비중) × 기준총액 ÷ 현재가. 정수로 반올림, 범위 비중(예: 3~5%)이면 주수도 범위(+N~M주)로.
- **매도/축소**: 매도하면 기준총액도 줄어드는 것을 반영해 매도금액 S를 (현재평가액 − S) = 목표비중 × (기준총액 − S)로 풀고, S ÷ 현재가를 정수화해 "−N주"로 표기.
- 표 컬럼: `분류 | 티커 | 비중 제안 | 주수 환산 | 근거`. 주수 환산 옆에 대략의 소요/회수 금액($)도 병기.
- 1주 단가가 큰 종목(예: BRK.B)은 반올림 때문에 목표비중과 어긋날 수 있다 — 실제 도달 비중을 괄호로 명시한다.
- 주문 실행은 절대 하지 않는다 (이 스킬에 주문 API는 없다). 주수는 계산상 제안일 뿐임을 보고서에 명시.

## 4단계 — 저장·완료 보고

- 보고서 저장: `docs/report/{오늘날짜 YYYY-MM-DD}/포트폴리오/포트폴리오_조언.md`
- 구성: 요약(3~5줄) → 포트폴리오 현황 표(티커·수량·평단·현재가·손익률·비중) → 구조 분석 → 4분류 조언 표 → 신규 분석 권고 목록 → 면책 문구
- 잔고 원본 JSON은 저장하지 않는다 (계좌 정보이므로 보고서에는 비중·손익률 등 파생 수치 위주로, 필요 최소한만 기재).
- 말미 면책 문구: "본 보고서는 개인 참고용 시뮬레이션이며 투자 자문이 아닙니다. 모든 투자 판단과 책임은 본인에게 있습니다."
- 사용자에게 완료 보고: 총자산·손익 요약 + 4분류 핵심(각 1줄) + 신규 분석 권고 티커 + 저장 경로.

## 주의사항

- `.env`, `.kis-token.json`은 반드시 .gitignore에 있어야 한다. 커밋 전 스테이징에 포함됐는지 확인하지 말고, 애초에 add하지 마라.
- API 명세 원문이 필요하면 `docs/koreainvestment/tokenP.txt`(토큰), `docs/koreainvestment/balance.txt`(잔고)를 읽는다. 스크립트 수정 시에만 필요하다.
- 계좌에 있지만 holdings ticker.md에 없는 종목(또는 그 반대)이 있으면 사용자에게 알려준다 — 수동 목록의 최신화 신호다.
