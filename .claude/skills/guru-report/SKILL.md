# Guru Report — 13인의 거장 투자 보고서

티커 하나를 입력받아 3단계 파이프라인으로 보고서를 생산한다:
**① 공용 데이터 팩 → ② 13인 개별 보고서 (Sonnet 병렬, 웹 조사 금지) → ③ 종합 최종 보고서 (세션 모델)**

토큰 절약 원칙: 외부 데이터 수집은 1단계에서 **딱 한 번**, 13인 에이전트는 **Sonnet**으로, 3단계는 13개 보고서 전문 대신 **요약 파일 + 선택 정독**으로 작동한다.

## 사전 확인

- `references/personas/` 폴더의 페르소나 파일 목록을 먼저 확인한다 (Glob: `references/personas/*.md`). 이 파일들이 13인의 거장 정의다. 개수가 `docs/books/`의 책 수와 다르면, 누락된 책에 대해 페르소나를 재생성한 뒤 진행한다 (아래 "페르소나 재생성" 참조).
- 날짜 폴더는 오늘 날짜 `YYYY-MM-DD` 형식. 모든 산출물 경로의 기준: `docs/report/{날짜}/`
- 티커가 모호하면(예: 종목명만 준 경우) 가장 유력한 티커로 해석해 진행하고, 보고서 헤더에 해석 근거를 남긴다.

## 1단계 — 공용 데이터 팩 작성

13인이 **동일한 사실관계** 위에서 판단해야 하고, 2단계 에이전트는 웹 조사가 금지되므로, 필요한 데이터는 여기서 전부 수집해 파일로 저장한다.

- 저장 위치: `docs/report/{날짜}/_data/{티커}.md`
- 수집 방법 (우선순위 순):
  1. **금융 데이터 MCP가 연결돼 있으면** (Daloopa, S&P Global/kFinance, Morningstar, FactSet 등 — ToolSearch로 확인) 그 도구를 우선 사용한다. 이때 `equity-research` 등 설치된 financial-services 스킬의 워크플로를 따라도 좋다.
  2. **MCP가 없으면** `references/data-sources.md`의 무료 엔드포인트(Yahoo Finance chart/fundamentals-timeseries/quoteSummary, SEC EDGAR)로 시세·재무·밸류에이션·컨센서스를 수집한다. 가입/API 키 불필요, 검증된 curl 레시피가 문서에 있다.
  3. 뉴스·이벤트·정성 정보는 WebSearch/WebFetch로 보완한다 (이 단계에서만 허용).
- 산출물은 마크다운 데이터 팩 하나면 충분하다 (Excel 모델 불필요). 표 구성은 `equity-research` 스킬의 출력 포맷(컨센서스 표 — 애널리스트 수·분산 포함, 재무 요약 표, 밸류에이션 표)을 따른다.
- 반드시 포함할 항목:
  - 기업 개요: 사업 모델, 주요 제품/세그먼트, 경쟁사
  - 재무 (최근 3~5년 + 최근 분기): 매출, 매출총이익률, 영업이익률, 순이익, EPS, FCF, 부채/자본, ROE, ROIC
  - **재무 안전성**: 유동자산·유동부채·유동비율, 총부채, 재고, 이자보상배율(산출 가능 시) — 그레이엄류 NCAV·안전성 평가용
  - 밸류에이션: 현재가, 시가총액, PER, PBR, PSR, EV/EBITDA, PEG, 배당수익률 (가능한 것 위주)
  - **수급·심리**: 공매도 비율(short % of float), 내부자·기관 지분율, 애널리스트 등급 분포(강력매수~매도 인원)와 목표가
  - 주가 흐름: 52주 고저와 현재 위치, 최근 1년 추이 요약, 최근 급등락 이벤트와 그 원인
  - 최근 뉴스/이벤트: 실적 발표 요지, 가이던스, 소송·규제·회계 이슈, 차기 실적 발표일 등 예정 이벤트 (출처와 날짜 명시)
  - 애널리스트 컨센서스가 확인되면 포함
- 확인 불가한 수치는 추정하지 말고 "확인 불가"로 표기한다. 모든 수치에 기준 시점을 적는다.

## 2단계 — 13인 개별 보고서 (Sonnet 병렬 에이전트)

페르소나 파일 하나당 에이전트 하나를 **한 메시지에서 전부 병렬로** 띄운다 (Agent 도구, 백그라운드, **`model: "sonnet"` 지정**). 각 에이전트에게 줄 프롬프트 구성:

1. 자신의 페르소나 파일 절대경로 (필독 지시)
2. 데이터 팩 절대경로 (필독 지시)
3. 저장 경로: `docs/report/{날짜}/{인물명}/{티커}.md` (인물명 = 페르소나 파일명에서 확장자 뺀 것, 폴더는 에이전트가 생성)
4. 아래 보고서 템플릿과 요약 블록 형식

에이전트 지시사항에 반드시 포함할 것:
- "너는 {인물명}이다. 페르소나 파일의 철학·체크리스트·말투를 그대로 재현하라. 중립적 애널리스트 톤 금지."
- "WebSearch/WebFetch/Bash 등 외부 조사 도구 사용 금지. 읽어도 되는 것은 페르소나 파일과 데이터 팩 두 개뿐이다. 데이터 팩에 없는 수치는 조사하지 말고 '확인 불가'로 남기고, 그 불확실성을 판단에 반영하라."
- "결론은 반드시 매수/보유/매도/관망 중 하나 + 확신도(1~10)로 끝내라. 이 인물이라면 내렸을 결론을 내려라 — 애매하게 도망치지 마라."
- "최종 응답으로는 아래 '요약 블록'만 반환하라. 인사말·부연 설명 금지."

### 개별 보고서 템플릿

```markdown
# {티커} 투자 보고서 — {인물명} ({책 제목})
> 날짜: {날짜} | 투자의견: {매수/보유/매도/관망} | 확신도: {n}/10

## 한 줄 결론
## 내 철학에서 본 이 기업
## 체크리스트 평가
(페르소나의 체크리스트 항목별 ○/△/× + 근거 수치)
## 밸류에이션 판단
(그 인물의 방법론으로. 적정가 산출이 가능한 철학이면 적정가/매수가격대 제시)
## 리스크
## 최종 의견
(그 인물의 말투로, 인용구 활용 가능)
```

### 요약 블록 (에이전트의 최종 응답 형식)

```
인물: {인물명} | 의견: {매수/보유/매도/관망} | 확신도: {n}/10
핵심 논거:
- (수치를 포함한 불릿 2~3개)
적정가/매수 가격대: (그 인물의 방법론 기준. 산출하지 않는 철학이면 "산출 안 함")
트리거·재검토 조건: (무엇이 확인되면 판단이 바뀌는가 1~2개)
대표 발언: (그 인물 말투의 핵심 문장 1개 — 최종보고서 인용용)
```

13개 에이전트가 전부 완료되면, 메인 루프는 반환된 요약 블록 13개를 그대로 모아 `docs/report/{날짜}/_data/{티커}_요약.md` 한 파일로 저장한다 (재가공 금지, 인물 순서대로 이어붙이기만).

## 3단계 — 종합 최종 보고서

종합 에이전트 **1개**를 띄운다 (모델 지정 없음 = 세션 모델). 프롬프트에 포함할 것:

1. 필독 2개: `_data/{티커}_요약.md` (13인 요약) + `_data/{티커}.md` (데이터 팩)
2. 선택 정독 지시: "요약을 훑은 뒤 대표 강세 1~2인, 대표 약세·관망 1~2인, 논거가 독특한 1인을 골라 **최대 4개까지만** 개별 보고서 전문(`docs/report/{날짜}/{인물명}/{티커}.md`)을 읽어 인용과 논거를 보강하라. 13개 전부 읽지 마라."
3. 표결 집계는 요약 파일에서 직접 계산하라.
4. 산출물 **1개**: `docs/report/{날짜}/최종/{티커}_최종보고서.md`

### 최종 보고서 형식 (`{티커}_최종보고서.md`)

```markdown
# {티커} 최종 투자 보고서
> 날짜: {날짜} | 종합 의견: {매수/보유/매도/관망} | 표결: 매수 n·보유 n·매도 n·관망 n

## 종합 결론
(표결과 확신도를 가중해 도출. 다수결 기계적 적용이 아니라, 13인의 논거 중 가장 견고한 것 중심으로)
## 강세론 핵심 (대표 발언자와 논거)
## 약세론 핵심 (대표 발언자와 논거)
## 미해결 쟁점
(13인의 논거가 갈리는 지점 — 무엇이 확인되면 판단이 바뀌는지, 트리거·재검토 조건 종합)
## 실행 가이드

**신규 매수 — 분할 진입 가격대:**

| 가격대 | 근거 제시자 | 논리 |
|---|---|---|
| (높은 가격대부터 내림차순) | (그 가격을 제시한 인물명. 같은 대역이면 `·`로 묶는다) | (그 가격이 나온 계산 근거를 수치로 — 예: fwd PER 25배, DCF 기본 시나리오, EY 6% 역산) |

(위 표는 **필수**다. 요약 파일의 "적정가/매수 가격대" 항목을 전부 모아 가격 내림차순으로 배열하고, 같은 대역을 제시한 인물은 한 행에 묶는다. "산출 안 함" 철학의 인물은 표에서 빼고 아래 불릿에서 다룬다. 현재가가 어느 행에 해당하는지 표 바로 아래 한 줄로 밝힌다.)

- **현재가 진입을 원한다면**: (매수 의견 낸 거장들의 진입 방식 — 분할/적립식 여부, 포지션 크기)
- **트레이딩 접근 (슈웨거)**: (돌파·손절 가격과 사이징)
- **기존 보유자**: (보유/차익실현 기준)
- **손절·재검토 조건 (전 진영 공통 감시 목록)**: (①②③④ 형태로, 무엇이 확인되면 논거가 깨지는가)

## 13인 요약표
| 인물 | 의견 | 확신도 | 핵심 논거 한 줄 |
```

## 완료 보고

모든 파일 생성 후 사용자에게: 종합 의견 + 표결 결과 + 13인이 가장 크게 갈린 쟁점 1~2개 + 생성된 폴더 경로를 요약해준다.

## 페르소나 재생성

`docs/books/`에 책이 추가/변경되었을 때: 해당 책을 읽는 에이전트를 띄워 기존 페르소나 파일과 같은 형식(frontmatter: name/book/book_file, 섹션: 투자 철학 핵심/종목 평가 체크리스트/선호·회피 기업/매수·매도 판단 기준/리스크 관점/말투와 성격/대표 인용구)으로 `references/personas/`에 생성한다. 기존 파일 하나를 형식 예시로 함께 읽게 하라.

## 주의사항

- 이 보고서는 투자 참고 자료다. 최종보고서 말미에 "본 보고서는 서적 기반 시뮬레이션이며 투자 자문이 아닙니다" 한 줄을 넣는다.
- 13개 에이전트 중 일부가 실패하면 실패한 것만 재시도하고, 그래도 실패하면 종합은 완성된 보고서·요약만으로 진행하되 사용자에게 누락을 알린다.
- 같은 날 같은 티커를 다시 돌리면 기존 파일을 덮어쓴다 (버전 관리가 필요하면 사용자에게 확인).
