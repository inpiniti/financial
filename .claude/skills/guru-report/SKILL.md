# Guru Report — 13인의 거장 투자 보고서

티커 하나를 입력받아 3단계 파이프라인으로 보고서를 생산한다:
**① 공용 데이터 팩 → ② 13인 개별 보고서 (병렬, 웹 조사 금지) → ③ 종합 최종 보고서**

## 토큰 절약 구조 (이 스킬의 핵심 설계)

메인 루프(세션 모델 = Opus)는 **오케스트레이터**다. 실제 작업(데이터 수집·13인 보고서·종합)은 전부 전용 서브에이전트에 위임하고, 메인은 라우팅·조립·완료보고만 한다. 각 단계 서브에이전트는 `.claude/agents/`에 정의돼 있고 **model·effort·tools·maxTurns가 프론트매터에 고정**돼 있으므로, 스폰할 때 모델을 지정할 필요가 없다.

| 단계 | subagent_type | 모델 | effort | 도구 | 비고 |
|---|---|---|---|---|---|
| 1 데이터 수집 | `guru-data` | sonnet | medium | Bash·Web·Read·Write | curl·파싱을 메인 밖에서 → Opus 컨텍스트 청정 |
| 2 정성 거장(9인) | `guru-persona` | sonnet | **low** | Read·Write | 사고 토큰 최소, 외부조사 원천 차단 |
| 2 계량 거장(4인) | `guru-persona-quant` | sonnet | medium | Read·Write | DCF·마법공식·공식·손익비만 medium |
| 3 종합 | `guru-synthesis` | 세션(opus) | medium | Read·Write | 판단만 Opus, effort는 낮춤 |

**thinking 주의**: 서브에이전트는 세션의 extended thinking 상태를 상속한다(개별 지정 불가). 13인 병렬이라 세션 thinking이 켜져 있으면 사고 토큰이 13배로 불어난다. **이 스킬을 돌리기 전 세션의 thinking을 끄는 것을 권장**한다(effort는 위 표대로 프론트매터가 이미 낮춰 둔다). 메인 루프도 이 작업에선 깊은 사고가 필요 없으니 thinking을 끈 채 진행한다.

**금융 MCP 없음**: 이 환경엔 Daloopa/kFinance 등 금융 MCP가 없다. **ToolSearch로 MCP를 찾지 마라**(불필요한 스키마 로딩으로 토큰만 샌다). 1단계는 곧장 무료 Yahoo/EDGAR 경로를 쓴다.

## 사전 확인

- `references/personas/` 폴더의 페르소나 파일 목록을 먼저 확인한다 (Glob: `references/personas/*.md`). 이 파일들이 13인의 거장 정의다. 개수가 `docs/books/`의 책 수와 다르면, 누락된 책에 대해 페르소나를 재생성한 뒤 진행한다 (아래 "페르소나 재생성" 참조).
- 날짜 폴더는 오늘 날짜 `YYYY-MM-DD` 형식. 모든 산출물 경로의 기준: `docs/report/{날짜}/`
- 티커가 모호하면(예: 종목명만 준 경우) 가장 유력한 티커로 해석해 진행하고, 데이터 팩 헤더에 해석 근거를 남긴다.

## 1단계 — 공용 데이터 팩 작성 (guru-data에 위임)

13인이 **동일한 사실관계** 위에서 판단해야 하고 2단계 에이전트는 웹 조사가 금지되므로, 필요한 데이터를 여기서 전부 수집해 파일 하나로 저장한다. **메인 루프가 직접 curl/파싱하지 말고 `guru-data` 서브에이전트에 위임**한다(그래야 JSON 덤프가 Opus 컨텍스트를 오염시키지 않는다).

- `Agent` 도구, `subagent_type: "guru-data"` (모델·effort는 정의에 고정 — 지정 불필요). 스폰 프롬프트에 넘길 것:
  - 티커, 오늘 날짜
  - 저장 경로: `docs/report/{날짜}/_data/{티커}.md`
  - 레시피 파일 절대경로: `.claude/skills/guru-report/references/data-sources.md` (에이전트가 읽어 그대로 따른다)
- 에이전트가 데이터 팩을 저장하고 간결한 확인(저장 경로 + 핵심 수치 몇 개 + "확인 불가" 항목)만 반환하면, 메인 루프는 팩 전문을 다시 읽지 말고 그 확인만 신뢰한다. 필요 시 파일을 1회만 훑는다.
- 데이터 팩에 반드시 들어갈 항목·간결성 규칙은 `guru-data` 정의에 명시돼 있다. (기업 개요·재무·재무안전성·밸류에이션·수급심리·주가흐름·뉴스이벤트·컨센서스)

## 2단계 — 13인 개별 보고서 (병렬 서브에이전트)

페르소나 파일 하나당 에이전트 하나를 **한 메시지에서 전부 병렬로** 띄운다 (Agent 도구, 백그라운드). **모델·effort·도구·템플릿·요약블록 형식은 서브에이전트 정의에 이미 들어 있으므로, 스폰 프롬프트는 페르소나별 차이만 짧게 담는다** (프롬프트를 13배로 부풀리지 않기 위함).

### subagent_type 라우팅

- **계량 방법론이 핵심인 4인 → `guru-persona-quant`** (effort medium):
  - `애스워스-다모다란` (DCF 시나리오별 주당가치 + 역산 DCF)
  - `조엘-그린블라트` (마법공식: 자본수익률 + 이익수익률 EBIT/EV)
  - `벤저민-그레이엄` (내재가치 공식 V=EPS×(8.5+2g), NCAV)
  - `잭-슈웨거` (진입·손절·목표·사이징·손익비 숫자로)
- **나머지 9인 → `guru-persona`** (effort low):
  - `워런-버핏`, `필립-피셔`, `피터-린치`, `세스-클라먼`, `모니시-파브라이`, `존-템플턴`, `마이클-버리`, `앙드레-코스톨라니`, `뉴욕주민`

### 스폰 프롬프트에 담을 것 (인물별로 이것만)

1. "너는 {인물명}이다." + 페르소나 파일 절대경로 (`references/personas/{인물명}.md`)
2. 데이터 팩 절대경로 (`docs/report/{날짜}/_data/{티커}.md`)
3. 저장 경로: `docs/report/{날짜}/{인물명}/{티커}.md` (폴더는 에이전트가 생성)
4. 티커, 날짜
5. (계량 거장만) 수행할 계산 지시 — 위 라우팅 괄호 안의 방법론
6. (린치·코스톨라니처럼 특유 프레임이 있으면) 한 줄 지시 — 예: 린치 "6가지 분류 중 어디인지(경기순환주 여부) 명시", 코스톨라니 "달걀 모형 국면 판정"

> 공통 규칙(두 파일만 병렬로 읽기, 외부조사 금지, 매수/보유/매도/관망+확신도로 마무리, 요약블록만 반환, 보고서 템플릿)은 **서브에이전트 정의에 이미 있으니 스폰 프롬프트에 반복하지 마라.**

### 요약 파일 취합

13개 에이전트가 반환한 요약 블록(형식은 정의에 명시됨)을 그대로 모아 `docs/report/{날짜}/_data/{티커}_요약.md` 한 파일로 저장한다 (재가공 금지, 인물 순서대로 이어붙이기만).

## 3단계 — 종합 최종 보고서 (guru-synthesis에 위임)

종합 에이전트 **1개**를 띄운다. `subagent_type: "guru-synthesis"` (model 미지정 = 세션 모델, effort는 정의에서 medium). 스폰 프롬프트에 담을 것:

1. 티커, 날짜
2. 필독 2개 경로: `_data/{티커}_요약.md` + `_data/{티커}.md`
3. (선택) 표결이 한쪽으로 크게 쏠렸으면 그 사실을 힌트로 — 예: "이번엔 매수 0명이니 강세론은 '기업의 질'을 인정한 논거로 구성하라"

> **저장 경로는 스폰 프롬프트에 넣지 마라.** 최종 산출물 경로는 `guru-synthesis` 정의에 `docs/report/{날짜}/최종/{티커}_최종보고서.md`로 고정돼 있다. 프롬프트에 다른 경로(예: `.../최종/{티커}.md`)를 적으면 에이전트가 정의 대신 그것을 따라 파일명이 틀어진다.
> 최종 보고서 형식·선택정독 규칙(최대 4개)·필수 진입가격대 표·면책 문구도 **guru-synthesis 정의에 이미 있다.**

## 완료 보고

모든 파일 생성 후 사용자에게: 종합 의견 + 표결 결과 + 13인이 가장 크게 갈린 쟁점 1~2개 + 생성된 폴더 경로를 요약해준다.

## 페르소나 재생성

`docs/books/`에 책이 추가/변경되었을 때: 해당 책을 읽는 에이전트를 띄워(`subagent_type: "guru-persona"`가 아니라 일반 에이전트로) 기존 페르소나 파일과 같은 형식(frontmatter: name/book/book_file, 섹션: 투자 철학 핵심/종목 평가 체크리스트/선호·회피 기업/매수·매도 판단 기준/리스크 관점/말투와 성격/대표 인용구)으로 `references/personas/`에 생성한다. 기존 파일 하나를 형식 예시로 함께 읽게 하라. 새 인물이 계량 방법론 중심이면 위 2단계 라우팅의 quant 목록에 추가한다.

## 주의사항

- 이 보고서는 투자 참고 자료다. 최종보고서 말미의 "본 보고서는 서적 기반 시뮬레이션이며 투자 자문이 아닙니다" 문구는 guru-synthesis가 넣는다.
- 13개 에이전트 중 일부가 실패하면 실패한 것만 재시도하고, 그래도 실패하면 종합은 완성된 보고서·요약만으로 진행하되 사용자에게 누락을 알린다.
- 같은 날 같은 티커를 다시 돌리면 기존 파일을 덮어쓴다 (버전 관리가 필요하면 사용자에게 확인).
- 서브에이전트 정의(`.claude/agents/guru-*.md`)를 수정하면 model/effort/도구 정책이 바뀐다. 품질이 부족하면 해당 단계의 `effort`를 한 단계 올리고, 더 아끼려면 내린다.
