# Guru Report — 13인의 거장 투자 보고서

티커 하나를 입력받아 3단계 파이프라인으로 보고서를 생산한다:
**① 공용 데이터 팩 → ② 13인 개별 보고서 (병렬) → ③ 원탁 토론 + 최종 보고서**

## 사전 확인

- `references/personas/` 폴더의 페르소나 파일 목록을 먼저 확인한다 (Glob: `references/personas/*.md`). 이 파일들이 13인의 거장 정의다. 개수가 `docs/books/`의 책 수와 다르면, 누락된 책에 대해 페르소나를 재생성한 뒤 진행한다 (아래 "페르소나 재생성" 참조).
- 날짜 폴더는 오늘 날짜 `YYYY-MM-DD` 형식. 모든 산출물 경로의 기준: `docs/report/{날짜}/`
- 티커가 모호하면(예: 종목명만 준 경우) 가장 유력한 티커로 해석해 진행하고, 보고서 헤더에 해석 근거를 남긴다.

## 1단계 — 공용 데이터 팩 작성

13인이 **동일한 사실관계** 위에서 논쟁해야 하므로, 데이터 수집은 딱 한 번 수행하고 결과를 파일로 저장한다.

- 저장 위치: `docs/report/{날짜}/_data/{티커}.md`
- 수집 방법 (우선순위 순):
  1. **금융 데이터 MCP가 연결돼 있으면** (Daloopa, S&P Global/kFinance, Morningstar, FactSet 등 — ToolSearch로 확인) 그 도구를 우선 사용한다. 이때 `equity-research` 등 설치된 financial-services 스킬의 워크플로를 따라도 좋다.
  2. **MCP가 없으면** `references/data-sources.md`의 무료 엔드포인트(Yahoo Finance chart/fundamentals-timeseries/quoteSummary, SEC EDGAR)로 시세·재무·밸류에이션·컨센서스를 수집한다. 가입/API 키 불필요, 검증된 curl 레시피가 문서에 있다.
  3. 뉴스·이벤트·정성 정보는 WebSearch/WebFetch로 보완한다.
- 산출물은 마크다운 데이터 팩 하나면 충분하다 (Excel 모델 불필요). 표 구성은 `equity-research` 스킬의 출력 포맷(컨센서스 표 — 애널리스트 수·분산 포함, 재무 요약 표, 밸류에이션 표)을 따른다.
- 반드시 포함할 항목:
  - 기업 개요: 사업 모델, 주요 제품/세그먼트, 경쟁사
  - 재무 (최근 3~5년 + 최근 분기): 매출, 매출총이익률, 영업이익률, 순이익, EPS, FCF, 부채/자본, ROE, ROIC
  - 밸류에이션: 현재가, 시가총액, PER, PBR, PSR, EV/EBITDA, PEG, 배당수익률 (가능한 것 위주)
  - 주가 흐름: 52주 고저, 최근 1년 추이 요약
  - 최근 뉴스/이벤트: 실적 발표, 가이던스, 주요 리스크 이슈 (출처와 날짜 명시)
  - 애널리스트 컨센서스가 확인되면 포함
- 확인 불가한 수치는 추정하지 말고 "확인 불가"로 표기한다. 모든 수치에 기준 시점을 적는다.

## 2단계 — 13인 개별 보고서 (병렬 에이전트)

페르소나 파일 하나당 에이전트 하나를 **한 메시지에서 전부 병렬로** 띄운다 (Agent 도구, 백그라운드). 각 에이전트에게 줄 프롬프트 구성:

1. 자신의 페르소나 파일 절대경로 (필독 지시)
2. 데이터 팩 절대경로 (필독 지시)
3. 저장 경로: `docs/report/{날짜}/{인물명}/{티커}.md` (인물명 = 페르소나 파일명에서 확장자 뺀 것, 폴더는 에이전트가 생성)
4. 아래 보고서 템플릿

에이전트 지시사항에 반드시 포함할 것:
- "너는 {인물명}이다. 페르소나 파일의 철학·체크리스트·말투를 그대로 재현하라. 중립적 애널리스트 톤 금지."
- "데이터 팩에 없는 수치가 필요하면 웹에서 조사하되, 조사 불가 시 '확인 불가'로 남겨라."
- "결론은 반드시 매수/보유/매도/관망 중 하나 + 확신도(1~10)로 끝내라. 이 인물이라면 내렸을 결론을 내려라 — 애매하게 도망치지 마라."
- "최종 응답으로는 다음 한 줄만 반환하라: `인물명 | 투자의견 | 확신도 | 핵심 논거 한 문장`"

### 개별 보고서 템플릿

```markdown
# {티커} 투자 보고서 — {인물명} ({책 제목})
> 날짜: {날짜} | 투자의견: {매수/보유/매도/관망} | 확신도: {n}/10

## 한 줄 결론
## 내 철학에서 본 이 기업
## 체크리스트 평가
(페르소나의 체크리스트 항목별 ○/△/× + 근거 수치)
## 밸류에이션 판단
(그 인물의 방법론으로. 적정가 산출이 가능한 철학이면 적정가/매수가격대 제시)
## 리스크
## 최종 의견
(그 인물의 말투로, 인용구 활용 가능)
```

## 3단계 — 원탁 토론 + 최종 보고서

2단계 에이전트가 전부 완료되면(완료 알림 수신), 토론 진행자 에이전트 **1개**를 띄운다. 진행자 프롬프트에 포함할 것:

1. 13개 개별 보고서의 절대경로 전부 (필독 지시)
2. 데이터 팩 경로
3. 산출물 2개:
   - `docs/report/{날짜}/최종/{티커}_토론.md` — 토론 전문
   - `docs/report/{날짜}/최종/{티커}_최종보고서.md` — 최종 보고서

### 토론 형식 (`{티커}_토론.md`)

- **라운드 1 — 입장 표명**: 13인 각자 자기 보고서 요지를 2~3문장으로 발언 (각자의 말투로)
- **라운드 2 — 쟁점 논쟁**: 보고서들 사이에서 실제로 갈리는 쟁점 2~4개를 추출해, 쟁점마다 찬반 거장들이 서로 직접 반박하게 한다. 각 발언은 그 인물의 철학과 보고서 내용에 근거해야 하며, 새로운 사실을 지어내지 않는다.
- **라운드 3 — 최종 입장**: 토론 후 입장을 바꾼 인물이 있으면 이유와 함께 명시. 최종 표결 집계 (매수 n / 보유 n / 매도 n / 관망 n).

### 최종 보고서 형식 (`{티커}_최종보고서.md`)

```markdown
# {티커} 최종 투자 보고서
> 날짜: {날짜} | 종합 의견: {매수/보유/매도/관망} | 표결: 매수 n·보유 n·매도 n·관망 n

## 종합 결론
(표결과 확신도를 가중해 도출. 다수결 기계적 적용이 아니라, 논쟁에서 살아남은 논거 중심으로)
## 강세론 핵심 (대표 발언자와 논거)
## 약세론 핵심 (대표 발언자와 논거)
## 미해결 쟁점
(토론으로 결판나지 않은 것 — 무엇이 확인되면 판단이 바뀌는지)
## 실행 가이드
(매수 시 진입 가격대/분할 전략, 손절·재검토 조건 — 해당 의견을 낸 거장들의 기준 인용)
## 13인 요약표
| 인물 | 의견 | 확신도 | 핵심 논거 한 줄 |
```

## 완료 보고

모든 파일 생성 후 사용자에게: 종합 의견 + 표결 결과 + 가장 뜨거웠던 쟁점 1~2개 + 생성된 폴더 경로를 요약해준다.

## 페르소나 재생성

`docs/books/`에 책이 추가/변경되었을 때: 해당 책을 읽는 에이전트를 띄워 기존 페르소나 파일과 같은 형식(frontmatter: name/book/book_file, 섹션: 투자 철학 핵심/종목 평가 체크리스트/선호·회피 기업/매수·매도 판단 기준/리스크 관점/말투와 성격/대표 인용구)으로 `references/personas/`에 생성한다. 기존 파일 하나를 형식 예시로 함께 읽게 하라.

## 주의사항

- 이 보고서는 투자 참고 자료다. 최종보고서 말미에 "본 보고서는 서적 기반 시뮬레이션이며 투자 자문이 아닙니다" 한 줄을 넣는다.
- 13개 에이전트 중 일부가 실패하면 실패한 것만 재시도하고, 그래도 실패하면 토론은 완성된 보고서만으로 진행하되 사용자에게 누락을 알린다.
- 같은 날 같은 티커를 다시 돌리면 기존 파일을 덮어쓴다 (버전 관리가 필요하면 사용자에게 확인).
